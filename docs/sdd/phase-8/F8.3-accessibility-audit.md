# F8.3 - Accessibility Audit

## Overview

WCAG 2.1 AA compliance toolkit providing automated checks for color contrast, heading hierarchy, form labels, ARIA attributes, keyboard navigation, and accessible color palettes.

## Architecture

```
Client/lib/accessibility/
  wcag-checker.ts       # Color contrast, heading hierarchy, form labels, landmarks
  keyboard-nav.ts       # Focus traps, arrow navigation, screen reader announcements
  aria-helpers.ts       # ARIA role/state props, tree item attributes, validation
  color-palette.ts      # Palette validation, accessible alternative suggestions
  accessibility.test.ts # 114 tests covering all modules
```

## Modules

| Module | Purpose |
|--------|---------|
| WCAG Checker | Contrast ratio calculation (WCAG 2.0 luminance), heading hierarchy validation, image alt text checks, form label checks, landmark region validation, markdown report generation |
| Keyboard Nav | Focus trap create/destroy, arrow key navigation with wrapping, element visibility checks, tab order sorting, screen reader announcements via aria-live |
| ARIA Helpers | Role-aware attribute generation, tree item props (critical for spec tree UI), dialog/tab panel/live region helpers, attribute validation catching invalid role-attribute combinations |
| Color Palette | SpecTree color definitions, palette-wide AA/AAA validation, HSL-based accessible alternative suggestion, full palette generation meeting AA contrast |

## Key Thresholds

| Check | AA Threshold | AAA Threshold |
|-------|-------------|---------------|
| Normal text contrast | 4.5:1 | 7:1 |
| Large text contrast | 3:1 | 4.5:1 |

## Test Coverage

114 tests: wcag-checker (38), keyboard-nav (24), aria-helpers (24), color-palette (18).
