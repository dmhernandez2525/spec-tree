# F10.3 - Mobile Responsive & PWA

## Overview

Progressive Web App configuration and responsive design utilities providing manifest generation, feature detection, caching strategies, breakpoint-based layouts, fluid typography, and WCAG-compliant touch targets.

## Architecture

```
Client/lib/pwa/
  pwa-config.ts       # Manifest generation, feature detection, caching, install status
  responsive-utils.ts # Breakpoints, device detection, columns, font clamping, touch targets
  pwa.test.ts         # 64 tests
```

## Modules

### PWA Config

- **Manifest Generation**: Default SpecTree manifest with override merging for name, colors, display mode, and icons
- **Feature Detection**: Checks browser support for service workers, manifest link, and Notification API
- **Notification Permissions**: Async permission request with granted/denied/unsupported/error states
- **Cache Strategies**: Named strategy definitions for stale-while-revalidate, cache-first, and network-first
- **Install Prompt**: Detects standalone mode (already installed), beforeinstallprompt support (available), or unsupported

### Responsive Utils

- **Breakpoints**: Tailwind CSS-aligned thresholds (xs:0, sm:640, md:768, lg:1024, xl:1280, 2xl:1536)
- **Device Detection**: `isMobile` (<768), `isTablet` (768-1023), `isDesktop` (>=1024)
- **Column Layout**: Responsive grid columns per breakpoint (xs/sm:1, md:2, lg:3, xl/2xl:4)
- **Fluid Typography**: Linear interpolation between sm and xl breakpoints, clamped to min/max font sizes
- **Touch Targets**: WCAG 2.5.5 compliant sizing; 44px for mobile (xs/sm), 36px for tablet and desktop
